# 설유화(sullyuwha) — 한옥 컨셉 디자인 시스템

> 전통 한복 브랜드 **설유화**의 홍보 페이지 디자인 가이드.
> 한옥의 절제된 아름다움 — 한지(韓紙)의 결, 먹(墨)의 깊이, 단청(丹靑)의 은은한 색, 창살(格子)의 리듬 — 을 현대적 웹 언어로 옮긴다.

---

## 1. 디자인 철학

한옥은 **비움과 여백**, **자연 재료의 물성**, **절제된 색채**로 완성된다.
설유화 페이지는 이 세 가지를 화면에 그대로 가져온다.

| 한옥의 원리 | 웹 디자인 번역 |
| --- | --- |
| 여백(餘白)의 미 | 넉넉한 `padding`/`margin`, 콘텐츠 사이 충분한 호흡 |
| 한지의 물성 | 따뜻한 오프화이트 배경, 미세한 종이 결 텍스처 |
| 먹의 깊이 | 순검정 대신 따뜻한 먹색(`#2B2622`) 사용 |
| 단청의 색 | 채도를 낮춘 적·청·황을 포인트로만 절제 사용 |
| 창살의 리듬 | 격자 레이아웃, 얇은 구분선, 반복 패턴 모티프 |
| 처마의 곡선 | 완만한 `border-radius`, 느린 이징 곡선의 모션 |

**핵심 원칙**
1. **절제** — 색은 포인트로만. 화면의 80%는 한지색/먹색.
2. **여백** — 요소를 채우기보다 비운다. 섹션 상하 여백은 관대하게.
3. **물성** — 플랫하되 종이·나무·먹의 질감을 은은하게 암시.
4. **위계** — 명조(明朝) 계열 제목 + 고딕 본문으로 전통과 가독성을 동시에.

---

## 2. 색상 팔레트 (오방색 기반)

오방색(五方色: 청·적·황·백·흑)을 현대적으로 **채도를 낮춰** 재해석한다.
순색 그대로 쓰면 촌스럽다 — 흙과 안료가 바랜 듯한 톤을 쓴다.

### 기본(Neutral) — 화면의 뼈대

| 토큰 | HEX | 이름 | 용도 |
| --- | --- | --- | --- |
| `--color-hanji` | `#FAF6EC` | 한지 | 기본 배경 |
| `--color-hanji-deep` | `#F2EAD8` | 진한 한지 | 카드/교차 섹션 배경 |
| `--color-meok` | `#2B2622` | 먹 | 기본 텍스트, 다크 섹션 배경 |
| `--color-meok-soft` | `#5A524A` | 옅은 먹 | 보조 텍스트 |
| `--color-giwa` | `#3A3D42` | 기와 | 진한 배경, 푸터 |
| `--color-line` | `#DED3BC` | 창호 선 | 구분선/보더 |

### 강조(Accent) — 단청에서 절제 추출

| 토큰 | HEX | 이름 | 용도 |
| --- | --- | --- | --- |
| `--color-dancheong-red` | `#9E3B36` | 단청 적 | 주요 강조, CTA, 브랜드 시그니처 |
| `--color-dancheong-blue` | `#33566E` | 단청 청 | 보조 강조, 링크 호버 |
| `--color-hwangto` | `#C2A15B` | 황토/금 | 디테일, 아이콘, 라인 장식 |
| `--color-solip` | `#4A5D4E` | 솔잎 | 자연 톤 보조 |

> **사용 비율 가이드**: 한지/먹 계열 **85%**, 황토 **8%**, 단청 적/청 **7%**.
> 강조색은 "여기가 중요하다"를 말할 때만. 남발 금지.

### 다크 섹션 (Hero / Footer)

먹·기와색 배경 위에 한지색 텍스트. 단청 적을 시그니처 포인트로.

## 3. 타이포그래피

전통과 가독성의 균형. **제목은 명조(세리프)**, **본문은 고딕(산세리프)**.

| 역할 | 서체 | 굵기 | 비고 |
| --- | --- | --- | --- |
| Display / 제목 | `Noto Serif KR` | 500·700 | 명조의 붓끝 느낌, 전통성 |
| 본문 / UI | `Noto Sans KR` | 400·500·700 | 높은 가독성 |

### 타입 스케일 (모바일 → 데스크톱, `clamp`)

| 토큰 | 크기 | 용도 |
| --- | --- | --- |
| Display | `clamp(2.5rem, 6vw, 4.5rem)` | Hero 헤드라인 |
| H2 | `clamp(1.75rem, 3.5vw, 2.75rem)` | 섹션 제목 |
| H3 | `clamp(1.25rem, 2vw, 1.6rem)` | 카드 제목 |
| Body | `clamp(1rem, 1.1vw, 1.125rem)` | 본문 |
| Small | `0.875rem` | 캡션/메타 |

### 조판 규칙 (필수)

```css
/* 한글은 공백(어절) 기준으로 줄바꿈 — 단어 중간 끊김 방지 */
word-break: keep-all;
overflow-wrap: break-word;
```

- 제목 `line-height`: **1.3**, 본문: **1.8** (한글은 넉넉하게).
- 제목 자간(`letter-spacing`): `-0.02em` (명조는 살짝 조여야 정갈).
- 문단 최대 폭: **`65ch`** — 눈이 편한 한 줄 길이.

## 4. 여백 · 레이아웃

한옥의 여백을 위해 **8px 그리드**를 쓰되, 섹션 간격은 관대하게.

| 토큰 | 값 |
| --- | --- |
| 콘텐츠 최대폭 | `1200px` (`--container`) |
| 좌우 거터 | 모바일 `20px` / 데스크톱 `40px` |
| 섹션 상하 패딩 | `clamp(4rem, 10vw, 8rem)` |
| 요소 간 리듬 | `8 / 16 / 24 / 40 / 64 / 96 px` |

- 레이아웃 그리드는 **창살**처럼 정렬 — 12컬럼, gap `24px`.
- 이미지는 **문(門)틀** 은유: 얇은 보더 + 살짝의 안쪽 여백.

---

## 5. 모티프 · 텍스처

절제된 장식으로 한옥의 디테일을 암시한다. 과하면 안 된다.

- **창살(格子) 패턴** — 얇은 선(`--color-line`)의 격자. 섹션 배경에 아주 옅게(opacity 0.04~0.08) 또는 구분 요소로.
- **한지 텍스처** — 배경에 미세한 노이즈/결. `background` 그라디언트로 은은하게.
- **처마 라인** — 섹션 경계에 완만한 곡선 또는 얇은 이중선.
- **낙관(印) 포인트** — 단청 적색 정사각형 도장 모티프를 로고/CTA 악센트로.
- **모서리** — `border-radius`는 최소화(2~4px). 한옥은 직선이 기본, 곡선은 처마에만.

## 6. 컴포넌트 스타일 규칙

### 버튼
- **Primary**: 먹색 배경 / 한지색 글자, 호버 시 단청 적으로 전환.
- **Secondary(선)**: 투명 배경 / 먹색 얇은 보더(1px) / 먹색 글자.
- 패딩 `14px 28px`, `border-radius: 2px`, 자간 `0.02em`.
- 호버 트랜지션 `280ms ease` — 느리고 차분하게.

### 카드 (한복 컬렉션)
- 한지-딥 배경, 1px `--color-line` 보더.
- 이미지 상단, 아래 제목(명조) + 짧은 설명.
- 호버: 살짝 떠오름(`translateY(-4px)`) + 그림자 은은하게.

### 섹션 헤딩
- 상단에 작은 한자/한글 **부제(예: 布 · 옷감)** + 황토색 얇은 선.
- 아래 큰 명조 제목. 좌측 정렬 기본, Hero만 중앙.

### 이미지 (R2)
- 모든 이미지는 Cloudflare R2 공개 버킷 + 커스텀 도메인에서 서빙.
- `next/image`로 최적화, `remotePatterns`에 R2 도메인 등록.
- 로딩 전/미설정 시 **한지 그라디언트 플레이스홀더**로 우아하게 대체.

## 7. 모션

한옥의 시간은 느리게 흐른다. 모션은 **차분하고 절제**되게.

- 기본 이징: `cubic-bezier(0.22, 1, 0.36, 1)` (느린 감속).
- 진입 애니메이션: `opacity` + `translateY(16px)` → 0, `600ms`.
- 스크롤 등장은 한 번만(`once`), 과한 패럴랙스 지양.
- `prefers-reduced-motion` 존중 — 모션 최소화 대응 필수.

## 8. 반응형 브레이크포인트

| 이름 | 폭 | 대상 |
| --- | --- | --- |
| base | `< 640px` | 모바일 (1컬럼) |
| `sm` | `≥ 640px` | 큰 모바일 |
| `md` | `≥ 768px` | 태블릿 (2컬럼) |
| `lg` | `≥ 1024px` | 데스크톱 (3~4컬럼) |
| `xl` | `≥ 1280px` | 와이드 |

- **모바일 퍼스트**로 작성. 데스크톱에서 여백과 컬럼 확장.
- 터치 타깃 최소 `44px`. 헤더는 모바일에서 햄버거 → 전체화면 메뉴.
- 이미지는 `sizes` 속성으로 뷰포트별 적정 해상도 서빙.

## 9. 페이지 구조 (섹션 순서)

한복 브랜드의 이야기를 위→아래로 풀어낸다.

1. **Hero** — 브랜드 슬로건 + 대표 한복 이미지 (다크/먹 배경).
2. **브랜드 스토리** — 설유화의 철학, 이름의 의미.
3. **한복 컬렉션** — 대표 라인업(당의·철릭·활옷 등) 카드 그리드.
4. **제작 과정(장인정신)** — 옷감 선택 → 재단 → 바느질 → 완성 단계.
5. **갤러리** — 착장/디테일 사진 그리드(메이슨리 느낌).
6. **문의 · 오시는 길** — 연락처, 위치, 예약 안내 (다크 배경).
7. **푸터** — 로고, 저작권, SNS.

## 10. 접근성 · 품질 기준

- 명도 대비 **WCAG AA** 이상 (본문 4.5:1, 큰 글자 3:1).
- 모든 이미지 `alt` 필수, 장식 이미지는 `alt=""`.
- 키보드 포커스 링 표시(`--color-dancheong-blue`).
- 시맨틱 HTML(`<header> <main> <section> <footer>`), 섹션마다 `aria-labelledby`.
- `word-break: keep-all` 전역 적용으로 한글 어절 단위 줄바꿈 보장.
